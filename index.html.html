DirectDebit.cc  ·  Phase 8  ·  Updated June 2026  ·  3 Workflows

1Cancellation Initiation & Immediate Collection Impact
Trigger: Ops or merchant clicks Cancel in the mandate actions dropdown on SCR-MND-001  ·  Actors: Ops Team / Merchant · System · RPA
ℹ️ Cancellation can only be initiated from SCR-MND-001 (Mandate Management screen). Only the ops team and the merchant can trigger it. Once confirmed, cancellation is irreversible — there is no undo.

ℹ️ The system takes immediate action on all collections and related mandates the moment cancellation is confirmed. Every collection status is handled — including all Phase 7 recovery statuses.

ℹ️ If Phase 7 recovery had already created a replacement mandate for this mandate (Group 1 — NEW DD CASES: codes A, F, G, V, W), the replacement mandate is also cancelled automatically alongside the original (Option A).
⚠️ Cancellation is permanent and irreversible. Once confirmed, the process cannot be stopped or reversed from any screen.

WF 1b — Processing Payment & Pending Bank Response: What Happens After Bank Responds
ℹ️ When cancellation is initiated and a collection is in Processing Payment or Pending Bank Response status, the system cannot cancel the collection immediately — the collection is already with the RPA or the bank. Phase 6 processes the settlement file as normal, then once the bank responds the system applies cancellation logic.

Processing Payment and Pending Bank Response are treated identically in this sub-workflow — both mean the collection is already in flight.

2RPA Cancellation File Submission & Bank Response
Actors: RPA · Bank · System  ·  Fully automated · No human action required after initiation · Continues from WF 1
ℹ️ Once all active collections have been resolved (WF 1), the RPA submits the cancellation file to the bank. The bank either confirms or rejects it. Wrong timing is the only reason the bank rejects a cancellation — the system waits and resubmits automatically until the bank accepts.

ℹ️ If a replacement mandate was also cancelled (Group 1 recovery), the RPA submits a separate cancellation file for the replacement mandate following the same flow below.

Phase 7 replacement mandate rule (Group 1 — NEW DD CASES)

ScenarioWhat Phase 8 does
Phase 7 recovery created a replacement mandate for this mandate (codes A, F, G, V, W) AND the merchant/ops cancels the original mandate via Phase 8The replacement mandate is also cancelled automatically. A separate Cancellation Pending record is created for the replacement mandate. The same WF 2 bank submission flow runs for both mandates. Both must receive bank confirmation before each is marked Cancelled.
Phase 7 recovery is still in progress (payer has not yet signed the replacement mandate, or bank has not yet approved it)The in-progress collections are set to Failed immediately. The replacement mandate — if already created — is cancelled via Phase 8. If the replacement mandate was not yet created (payer had not clicked the link), no replacement mandate to cancel.
