Phase 8 Workflows — Mandate Cancellation & Confirmation

DirectDebit.cc  ·  Phase 8  ·  Updated June 2026  ·  3 Workflows

1Cancellation Initiation & Immediate Collection Impact
Trigger: Ops or merchant clicks Cancel in the mandate actions dropdown on SCR-MND-001  ·  Actors: Ops Team / Merchant · System · RPA
ℹ️ Cancellation can only be initiated from SCR-MND-001 (Mandate Management screen) by the ops team or the merchant. Once confirmed, cancellation is irreversible — there is no undo.

ℹ️ The moment cancellation is confirmed, the system immediately resolves every collection linked to this mandate. Each collection's current status determines what happens to it — see the summary table below.

⚠️ Cancellation is permanent and irreversible. Once confirmed, the process cannot be stopped or reversed from any screen.

WF 1b — Waiting for a Collection Result Before Proceeding with Cancellation
ℹ️ When cancellation is initiated and a collection is in Processing Payment or Pending Bank Response status, the system waits for the collection result to come back from the bank before it proceeds with the mandate cancellation. Once the result arrives — whether Received or Bounced — the mandate cancellation continues.

2RPA Cancellation File Submission, Bank Response & Payer Notification
Actors: RPA · Bank · System  ·  Fully automated · No human action required after initiation · Continues from WF 1
ℹ️ Once all active collections have been resolved (WF 1), the RPA submits the cancellation file to the bank. The bank either confirms or rejects it. Wrong timing is the only reason the bank rejects a cancellation — the system waits and resubmits automatically until the bank accepts.

ℹ️ The moment the mandate is confirmed Cancelled, the system automatically sends a cancellation notification to the payer via the configured contact channel (SMS, WhatsApp, or Email). This is sent for all cancellations — whether initiated by ops, merchant, or triggered automatically by Phase 7 Group 1 recovery.

Pending to be Replaced — how it works

StepWhat happens
1. Phase 7 bounces with codes A, F, G, V, WPhase 7 creates a new mandate for the payer and sets the original mandate status to "Pending to be Replaced" in SCR-MND-001. No Cancel action is shown for this mandate.
2. Phase 7 recovery completesThe payer signs the new mandate. The bank approves it. The new mandate is now active.
3. Original mandate enters Phase 8 automaticallySystem sets the original mandate to Cancellation Pending. Phase 8 normal flow continues (WF 1 → WF 2). Payer is notified when cancellation is confirmed.
4. If merchant wants to cancel the NEW mandateThe merchant must wait until the new mandate is issued and approved (step 2). Then they can cancel the new mandate like any other active mandate — the Cancel action is available on it normally.
